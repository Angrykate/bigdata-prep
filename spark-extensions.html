Extensions Spark & Pig

Spark ML pour le machine learning distribué à grande échelle, et Apache Pig pour les transformations ETL via un langage de script de haut niveau (Pig Latin).

Spark ML — Machine Learning distribué

La bibliothèque MLlib de Spark fournit des algorithmes ML qui s'exécutent en parallèle sur un cluster.

Spark ML vs scikit-learn — pourquoi choisir Spark ML ?

Critèrescikit-learnSpark ML
ExécutionMachine unique (single-node)Cluster distribué (multi-nœuds)
Volume de donnéesLimité par la RAM d'une machinePétaoctets — données sur HDFS/S3
Vitesse d'entraînementLente sur gros volumesParallélisée sur les executors

💡 À retenir : recommandez Spark ML à un ami dès que les données ne tiennent plus en RAM sur une seule machine, ou que le temps d'entraînement devient prohibitif.

Concepts fondamentaux de Spark ML

Transformer

Objet qui transforme un DataFrame en un autre via la méthode transform(). Ne s'entraîne pas — applique une règle déjà définie.

Ex : VectorAssembler, StringIndexer, StandardScaler, un modèle déjà entraîné (Model).

Estimator

Objet qui s'entraîne sur un DataFrame via la méthode fit() et produit un Transformer (le modèle entraîné).

Ex : LogisticRegression, RandomForest, KMeans, LinearRegression.

Pipeline

Chaîne ordonnée de Transformers et d'Estimators. Simplifie la gestion du workflow ML : prétraitement + entraînement en une seule commande fit().

Reproducible, testable, et facilement réutilisable.

VectorAssembler

Transformer incontournable : assemble plusieurs colonnes numériques en un seul vecteur dense features, format attendu par les algorithmes Spark ML.

ParamGrid & CrossValidator

Permettent le réglage des hyperparamètres : ParamGrid définit la grille de valeurs à tester, CrossValidator évalue chaque combinaison par validation croisée.

Évaluateurs

Mesurent la qualité du modèle sur des données de test.

BinaryClassificationEvaluator MulticlassClassificationEvaluator RegressionEvaluator

Pipeline Spark ML — exemple complet

Workflow type d'un TP Spark ML

from pyspark.ml import Pipeline
from pyspark.ml.feature import VectorAssembler, StringIndexer, StandardScaler
from pyspark.ml.classification import LogisticRegression
from pyspark.ml.evaluation import BinaryClassificationEvaluator

# 1. Charger les données
df = spark.read.option("header", True).option("inferSchema", True).csv("data.csv")
train, test = df.randomSplit([0.8, 0.2], seed=42)

# 2. Encoder la colonne cible (label)
indexer = StringIndexer(inputCol="categorie", outputCol="label")

# 3. Assembler les features en un vecteur
assembler = VectorAssembler(
  inputCols=["age", "revenu", "score"],
  outputCol="features_raw"
)

# 4. Normaliser
scaler = StandardScaler(inputCol="features_raw", outputCol="features")

# 5. Modèle
lr = LogisticRegression(featuresCol="features", labelCol="label")

# 6. Pipeline : tout assembler
pipeline = Pipeline(stages=[indexer, assembler, scaler, lr])

# 7. Entraîner (un seul fit() fait tout)
model = pipeline.fit(train)

# 8. Évaluer
predictions = model.transform(test)
evaluator = BinaryClassificationEvaluator(labelCol="label")
auc = evaluator.evaluate(predictions)
print(f"AUC : {auc:.3f}")

Ordre d'exécution dans le Pipeline

Lors du fit() sur les données d'entraînement :

StringIndexer.fit() → VectorAssembler.transform() → StandardScaler.fit() → LogisticRegression.fit()

Lors du transform() sur les données de test :

Chaque étape applique ses paramètres appris sur les nouvelles données, sans ré-entraînement.

Apache Pig

Outil de l'écosystème Hadoop pour les transformations ETL, via un langage de script de haut niveau : Pig Latin.

Qu'est-ce qu'Apache Pig ?

Plateforme de haut niveau pour le traitement de données volumineuses sur Hadoop. Pig Latin est un langage de flux de données (dataflow language) qui compile en jobs MapReduce. Développé par Yahoo!, maintenant Apache.

Orienté ETL Compile en MapReduce Pas de SQL Schéma flexible

  • MapReduce mode : sur un cluster Hadoop (production)

Exemple — WordCount en Pig Latin

-- Charger les données texte
lignes = LOAD 'hdfs://data/texte.txt' AS (ligne:chararray);

-- Découper chaque ligne en mots
mots = FOREACH lignes GENERATE FLATTEN(TOKENIZE(ligne)) AS mot;

-- Grouper par mot
groupes = GROUP mots BY mot;

-- Compter les occurrences
comptage = FOREACH groupes GENERATE group AS mot, COUNT(mots) AS nb;

-- Trier par fréquence décroissante
resultat = ORDER comptage BY nb DESC;

-- Sauvegarder
STORE resultat INTO 'hdfs://output/wordcount';

Exemple — Filtrage et agrégation

-- Charger un CSV avec schéma
ventes = LOAD 'hdfs://data/ventes.csv'
         USING PigStorage(',')
         AS (produit:chararray, region:chararray, montant:float);

-- Filtrer sur une région
ventes_nord = FILTER ventes BY region == 'Nord';

-- Grouper par produit
par_produit = GROUP ventes_nord BY produit;

-- Calculer le CA total
ca_produit = FOREACH par_produit GENERATE
             group AS produit,
             SUM(ventes_nord.montant) AS ca_total;

-- Afficher (debug)
DUMP ca_produit;

Pig — points importants à retenir

Lazy evaluation

Pig est également lazy : les transformations ne s'exécutent que lorsqu'une commande DUMP ou STORE est appelée.

UDF (User-Defined Functions)

Pig permet d'écrire des fonctions personnalisées en Java, Python ou JavaScript pour des transformations non couvertes par les fonctions intégrées.

FLATTEN

Décompresse une structure imbriquée (bag ou tuple) en lignes séparées. Indispensable après TOKENIZE ou GROUP pour travailler sur des éléments individuels.

Utilisation aujourd'hui

Pig est progressivement remplacé par Spark (plus rapide, en mémoire) et Hive (plus accessible). Il reste présent dans les pipelines ETL legacy dans les grandes entreprises.
